💭 为什么这个路线成立

前人（HDiT、ΔConvFusion、PiT）已经定义好了测什么 + 怎么测。你的工作不是在 DDPM 场景下重新发明测量方法，而是把这些方法搬到 Flow Matching 下重新做一遍，并且加一个关键维度——per-t 分析。如果发现 per-t ERF 有显著差异（比如 t=0.9 时 ERF 是 t=0.1 时的 2~3×），那就是一个没人报道过的新发现，直接通向 t-adaptive kernel 方法。

Per-t 分析的价值

直觉：Flow Matching 下 t 的语义很清晰——t→1 时 x_t 接近纯噪声，需要全局视野识别结构；t→0 时 x_t 接近真实图像，局部窗口即可。如果实验验证了这个直觉，就打开了一个新维度：generation process 的不同阶段应该用不同的 receptive field。这在 DDPM 文献中几乎没人分析过。

后续可做的方向

近期：(1) latent space 验证 — 拿 SD/Flux pretrained model hook attention 画 per-t ERF 曲线，零训练成本；(2) 256×256 扩展 — 验证结论可扩展性；(3) t-adaptive kernel 方法 — 离散调度或可学习调度。

中期：(1) 理论分析 — 从 flow ODE / score function Lipschitz 常数角度解释为什么 t 影响 optimal receptive field；(2) 跨架构验证 — UNet/DiT/AR 上系统测量；(3) 视频生成 — temporal attention 也有 per-step 行为差异。

长期：(1) 动态计算生成模型 — 不同 t 用不同架构配置（不光是 kernel size）；(2) attention 可解释性工具 — 把测量方法做成通用分析框架；(3) 高效生成架构设计 — learned sparsity / multi-scale heads / timestep-conditional routing。

核心 1 NA 在 pixel-space generation 上怎么验证的

1. HDiT: Scalable High-Resolution Pixel-Space Image Synthesis with Hourglass Diffusion Transformers
Katherine Crowson 等 · ICML 2024 · arxiv: 2401.11605
第一篇把 NA 用在 pixel-space 生成的论文。关键不是 hourglass 架构——是它的消融方法论（Table 1）：逐步替换组件，量化 FID + GFLOPs。你的 Phase 1 实验表格照搬这个结构。⚠️ 注意：架构贡献的是效率（O(n²)→O(n)），不是质量（128² 上 FID 净赚仅 ~1 点）；质量提升大头来自 Soft-Min-SNR loss。

✅ 必读（~3页）

  • Table 1 + Section 5.2 消融路线 — 怎么从 baseline 一步步加组件。你论文 Table 1 的模板。（注意：原文没有 Section 3.3，NA 相关内容在 §4.3 和 §5.2）
  • Section 4.3 Efficient Scaling — NA 放在高分辨率层，global attention 只放在最低分辨率层（16²），kernel size=7。
  • Appendix D.1 Table 5 — 完整消融（含被主表隐藏的 R2 和额外 baseline）。

📝 陪读笔记（2026-07-18）

🔍 阅读指南的勘误

  • Section 3.3 不存在 — 论文结构是 §3 Preliminaries（只有 3.1 Diffusion Models），NA 相关内容在 §4.3（仅 3 句话）和 §5.2（Table 1 消融）。
  • NA 为什么赢 Swin？原文没有理论分析 — 只有一句 "Neighborhood attention performs significantly better in practice" + Table 1 的数字。如果想知道深层原因，需要读 Hassani et al. (2023) NAT 原论文。
  • Kernel size 没有配置策略 — 全实验统一 kernel_size=7（Table 7），没有消融为什么选 7，没有讨论不同层该不该用不同 size。

🧪 Table 1 消融：到底每一步加了什么

Step 改了什么 白话 FID↓ GFLOPs
R1 DiT-B/4 原版（flat + pixel-space） flat 结构 + 全全局 attention 42.03 106
R3 R1 → HDiT block + trainer flat 但换 HDiT 的 block 设计 42.49 106
R4 R3 + Soft-Min-SNR 只改 loss，不改架构 30.71 106
A Flat → Hourglass（全 global attn） 加层级结构，Pixel-Unshuffle 下采样 50.76 32 (-70%)
B1 A + Swin Attention 高分辨率层换 Shifted Window 55.93 29
B2 A + Neighborhood Attention 高分辨率层换 NA（滑动窗口） 51.07 29
C B2 + GeGLU FFN 里 GeLU → GEGLU（数据门控） 44.36 31
D C + Axial RoPE 加性位置编码 → 2D 旋转位置编码 41.41 31
E D + Soft-Min-SNR 只改 loss，架构不变 27.74 31

🔑 关键发现

  • 架构贡献效率，loss 贡献质量：在 128² 上，架构改动（A→D）FID 只从 50.76→41.41（对比 R3 的 42.49 几乎没赚），但 GFLOPs 从 106→31（-70%）。loss 改动（D→E 或 R3→R4）各贡献 12-14 个 FID 点。真正值钱的是 O(n) 复杂度——512² 时 HDiT 比 DiT 便宜 32×，1024² 时便宜 100×，flat DiT 根本跑不动高分辨率。
  • R2 被隐藏了：R1→R3 中间实际有 R2（DiT 模型 + HDiT trainer，FID 崩到 69.86），因为 trainer 不兼容被从主表删掉、扔进 Appendix D.1。学术上不算造假但算选择性汇报。你论文可以学：失败的中间步骤放附录，主表保持干净。
  • R1 没有公开模型：只是拿开源 DiT 代码去掉 VAE 跑 pixel-space 得到的一行数字，两个 repo（DiT + k-diffusion）都在，可以复现。
  • NA vs Swin 精确定量：Swin（B1）FID 比纯 global（A）+5.17；NA（B2）只 +0.31。Swin 的窗口边界问题在生成任务上比判别任务严重得多。

🧮 Soft-Min-SNR 一句话

基于 Min-SNR（Hang et al. 2023, ICCV）的平滑改进：把硬截断 min(1/σ², γ) 换成 1/(σ² + 1/γ)，γ=4。直观：在分母加小常数 0.25，让低噪声时 loss 权重平滑逼近上限 4 而不是一刀切。HDiT 不是这个 idea 的原创者，只是做了平滑化 + 调参。

⚠️ 对你论文的影响

  • Table 1 消融格式可以直接照搬，但把 FID 贡献和计算效率贡献分两列报，比 HDiT 更诚实，审稿人会喜欢。
  • NA kernel size=7 是 magic number——你应该做 kernel size 消融（这是 HDiT 没做的事，可以是你的额外贡献）。
  • 在你的消融里明确区分「架构改动」和「训练策略改动」（loss weighting 不算架构），避免像 HDiT 一样让人误读。

⚠️ 核心风险

🔴 风险 1：per-t ERF 差异不显著——那你的核心增量就没了
最坏情况：t=0.9 和 t=0.1 的 ERF 几乎一样。那你唯一的增量就只是「在 FM 下复现了 DDPM 的结论」。这有 workshop/短文价值但冲不了高。防御：(1) 直觉很强——flow 路径更直，早期步需要从噪声里恢复结构，理论上需要更大视野；(2) 即使 per-t 不显著，NA 在 FM 下的系统性能分析 + FLOPs vs FID trade-off 本身也是贡献。
🟡 风险 2：被 scoop——per-t analysis 这个 idea 本身就挺自然的
t-dependent receptive field 这个直觉不难想到，可能在审稿中被已有工作覆盖。防御：(1) Phase 1 结果出来尽早 arXiv；(2) 不只是报告现象，要给出机制解释和 actionable insight。
🟡 风险 3：SiT 代码改不动 / fine-tune 不收敛
